## 1. 产品概述

同人合志发行工作台——面向同人社团的多人协作 Web 平台，让主催、画手、写手和校对从收稿到上线一站式完成合志制作与发行。解决传统合志制作中收稿混乱、校对无追踪、上线遗漏等痛点，面向同人社团、纪念本企划和活动合志团队。

## 2. 核心功能

### 2.1 用户角色

| 角色 | 加入方式 | 核心权限 |
|------|----------|----------|
| 主催 | 创建项目时自动成为 | 管理项目全部信息、分配篇目、确认校对、执行发布 |
| 画手 | 被主催邀请加入 | 上传插画/封面、查看自己负责的篇目、处理校对标注 |
| 写手 | 被主催邀请加入 | 上传文稿、查看自己负责的篇目、处理校对标注 |
| 校对 | 被主催邀请加入 | 预览全部篇目、标注错字/断页/出血线问题 |

### 2.2 功能模块

1. **工作台首页**：项目总览、我的任务、最近动态
2. **项目管理页**：刊物信息、参与人员、截稿日期、篇目列表、公开范围
3. **收稿上传页**：成员专属上传入口、文件状态追踪
4. **校对流转页**：预览标注、问题分类、处理确认
5. **发行确认页**：上线前检查清单、逐项核验、一键发布

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 工作台首页 | 项目卡片列表 | 展示用户参与的所有项目，显示进度百分比、截稿倒计时 |

## 3. 核心流程

### 3.1 项目创建与收稿流程

主催创建项目 → 填写刊物信息 → 邀请成员并分配角色 → 创建篇目并指定负责人 → 成员在专属入口上传稿件 → 主催查看上传状态（页数/尺寸/缺封面/未签授权）→ 督促进度

```mermaid
flowchart TD
    "主催创建项目" --> "填写刊物信息与截稿日"
    "填写刊物信息与截稿日" --> "邀请成员分配角色"
    "邀请成员分配角色" --> "创建篇目指定负责人"
    "创建篇目指定负责人" --> "成员上传稿件"
    "成员上传稿件" --> "主催查看上传状态"
    "主催查看上传状态" --> "全部收齐?"
    "全部收齐?" -->|"否"| "督促进度"
    "督促进度" --> "成员上传稿件"
    "全部收齐?" -->|"是"| "进入校对流转"
```

### 3.2 校对流转流程

校对员预览稿件 → 标注问题（错字/断页/出血线）→ 作者查看标注 → 修改并点击"已处理" → 主催统一确认 → 全部确认后进入发行准备

```mermaid
flowchart TD
    "校对员预览稿件" --> "标注问题"
    "标注问题" --> "作者查看标注"
    "作者查看标注" --> "修改稿件"
    "修改稿件" --> "点击已处理"
    "点击已处理" --> "主催统一确认"
    "主催统一确认" --> "全部确认?"
    "全部确认?" -->|"否"| "退回修改"
    "退回修改" --> "作者查看标注"
    "全部确认?" -->|"是"| "进入发行准备"
```

### 3.3 发行确认流程

主催打开发行清单 → 逐项检查（分级提示/试读范围/价格/收益分配/下架规则）→ 全部通过 → 解锁发布按钮 → 发布上线

```mermaid

    "检查分级提示" --> "设置试读范围"

    "填写价格" --> "确认收益分配说明"

    "确认下架规则" --> "全部通过?"

    "补充未完成项" --> "检查分级提示"
    "全部通过?" -->|"是"| "解锁发布按钮"
    "解锁发布按钮" --> "发布上线"
```

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深墨色（#1a1a2e）+ 朱砂红（#c0392b）点缀，营造和风印刷所质感
- **辅助色**：和纸米白（#f5f0e8）、靛蓝（#2c3e7b）、金箔色（#d4a843）
- **按钮风格**：圆角微凸，朱砂红主按钮 + 靛蓝次要按钮，悬停时微浮起
- **字体**：标题使用 Noto Serif SC 衬线体，正文使用 Noto Sans SC 无衬线体
- **布局风格**：侧边栏导航 + 卡片式内容区，大圆角卡片配合和纸纹理背景
- **图标风格**：线性图标搭配微小朱砂色圆点指示状态
- **装饰元素**：竖排文字标签、和风边框纹样、印章式状态标记

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|---------|

### 4.3 响应式设计

- 桌面优先设计，最小宽度 1024px
- 平板端侧边栏收起为汉堡菜单，卡片网格自动调整列数
- 移动端简化为单列布局，校对标注工具栏改为底部浮动栏

### 4.4 动效设计

- 页面加载：卡片交错淡入（stagger 100ms）
- 状态变更：印章标记盖下动画（scale + rotate）
- 校对标注：标记点脉冲发光动画
- 发布按钮解锁：从灰到朱砂的渐变填充动画
- 翻页：横向滑动过渡
